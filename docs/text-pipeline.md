# テキスト前処理パイプライン

パイプラインパターン。各ステップを共通インターフェースの関数として実装し、配列で順序管理。順序厳守。

## 適用順序

1. **リプライメンション除去**: `message.reference` 存在時、先頭の返信先ユーザーメンションを除去
2. **改行→句点変換**: 改行文字 `\n` を句点「。」に置換して読み上げ時のポーズを生成（すでに句読点がある場合は追加しない）
3. **コードブロック除去**: ` ```...``` ` → 「コードブロック省略」
4. **インラインコード除去**: `` `...` `` → 「コード省略」
5. **スポイラー除去**: `||...||` → 空文字
6. **URL 除去**: `https://...` → 「URL省略」
7. **タイムスタンプ変換**: `<t:1234567890:R>` 等をフォーマットスタイルに応じた日時文字列に変換
8. **カスタム絵文字**: サーバー設定により `<:name:123>` → 「name」or 除去
9. **ユーザーメンション変換**: `<@123>` → 「ユーザー名」(+ さん付け)
10. **ロールメンション変換**: `<@&123>` → 「ロール名」
11. **チャンネルメンション変換**: `<#123>` → 「チャンネル名」
12. **辞書変換**: サーバー辞書+グローバル辞書をマージ (サーバー辞書優先)、1つのトライ木 (最長一致) で1パス適用
13. **「w」「草」変換**: `w` → 「わら」、`ww`以上 → 「わらわら」(大文字W・全角ｗＷも同様)。`草` → 「くさ」
14. **英数字の読み方最適化**: カンマ区切り数字、英略語・単位変換 (kg→キログラム等)
15. **ローマ字読み**: サーバー設定 ON 時、ローマ字パターンをそのまま残す

## 後処理

- 最大文字数超過 → 「以下省略」付加してカット (FREE: 50、PREMIUM: 200 でクランプ)
- 前処理後にテキストが空 → 読み上げスキップ
- パイプライン完了後にスタンプ名・添付ファイル件数を末尾に付加

## 実装ルール

- 正規表現は全てモジュールレベルでプリコンパイル
- トライ木は VC 接続中のギルドのみメモリ保持、VC 切断時に破棄
- VC 接続後の初回辞書変換時に遅延構築 (lazy build)
- 辞書変更時は Redis Pub/Sub で無効化フラグを立て、再構築は次回アクセス時に遅延実行
- グローバル辞書変更時は全ギルドのトライ木を無効化

## パイプラインステップの共通インターフェース

各ステップは同じシグネチャの関数として実装する。ステップの追加・削除・並び替え・テストが容易な構成とする。
